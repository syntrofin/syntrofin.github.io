Revenue Recognition

Full Recognition (Point in Time)

  • How it creates the POB:        Single milestone; turns "Green" immediately upon triggering event (e.g., Invoice).
  • How it creates the Schedule:    Single RevenueTransaction for 100% of the value.
  • Accounting Result:           One SAP Document Number; no monthly "bleeding."

Daily/Monthly Proration (Over-Time)

  • How it creates the POB:        Set to "Fulfilled" on the Service Start Date.
  • How it creates the Schedule:    Uses Start/End Dates to calculate days/months ("Slices").
  • Accounting Result:            Multiple SAP Document Numbers (one per month).


Milestone-Based (Event-Based)

  • How it creates the POB:        Created in "Pending" (Red Light); waits for external updates like Task completion.
  • How it creates the Schedule:    Remains "Active" but dormant; generates records only on percentage changes.
  • Accounting Result:            SAP entries created only when business milestones are reached.

Manual Recognition

  • How it creates the POB:        Created without automated logic; stays "Not Started."
  • How it creates the Schedule:    Built, but requires manual user action (button click) to generate transactions.
  • Accounting Result:            Complete manual control over the SAP BAPI trigger.
